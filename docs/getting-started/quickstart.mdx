## What You'll Accomplish

By the end of this quickstart you'll have:

- Written a `uds-bundle.yaml` referencing two Zarf packages
- Created a bundle locally
- Deployed it to a Kubernetes cluster

To understand how bundles fit into the UDS ecosystem, see [Bundles](/concepts/configuration-and-packaging/bundles/).

---

## Prerequisites

- [UDS CLI installed](/cli/getting-started/installation/)
- A running Kubernetes cluster with `kubectl` configured (e.g., k3d, kind)

---

## Steps

<Steps>

1. **Create a `uds-bundle.yaml`**

   Create a new directory and add a `uds-bundle.yaml`:

   ```yaml title="uds-bundle.yaml"
   kind: UDSBundle
   metadata:
     name: my-first-bundle
     description: A minimal UDS bundle example
     version: 0.0.1

   packages:
     - name: init
       repository: ghcr.io/defenseunicorns/packages/init
       ref: v0.33.0
     - name: podinfo
       repository: ghcr.io/defenseunicorns/uds-cli/podinfo
       ref: 0.0.1
   ```

   This bundle deploys the Zarf init package followed by [podinfo](https://github.com/stefanprodan/podinfo).

2. **Create the bundle**

   From the directory containing your `uds-bundle.yaml`:

   ```bash
   uds create .
   ```

   UDS CLI pulls the referenced Zarf packages from their OCI registries and assembles them into a local tarball:

   ```text
   uds-bundle-my-first-bundle-amd64-0.0.1.tar.zst
   ```

   > [!NOTE]
   > To create directly into an OCI registry instead: `uds create . -o ghcr.io/<your-org>/dev`

3. **Deploy the bundle**

   ```bash
   uds deploy uds-bundle-my-first-bundle-amd64-0.0.1.tar.zst --confirm
   ```

   > [!TIP]
   > Omit `--confirm` to see a pre-deploy summary of all packages, overrides, and variables before confirming.

   To deploy from an OCI registry instead:

   ```bash
   uds deploy ghcr.io/<your-org>/dev/my-first-bundle:0.0.1 --confirm
   ```

</Steps>

---

## Verify

After deployment, confirm the pods are running:

```bash
uds zarf tools kubectl get pods -A
```

## Bundle Syntax Notes

**The `uds-bundle.yaml` is entirely declarative.** UDS CLI will not prompt for optional Zarf components at deploy time. To include an optional component, list it explicitly:

```yaml title="uds-bundle.yaml"
packages:
  - name: init
    repository: ghcr.io/defenseunicorns/packages/init
    ref: v0.33.0
    optionalComponents:
      - git-server
```

**Defense Unicorns shorthand** — when running `deploy`, `inspect`, `remove`, or `pull`, short paths automatically expand to the Defense Unicorns GHCR org:

```bash
# These are equivalent:
uds deploy unicorn-bundle:v0.1.0
uds deploy ghcr.io/defenseunicorns/packages/uds/bundles/unicorn-bundle:v0.1.0
```

## Troubleshooting

| Symptom | Check |
|---|---|
| `bundle architecture mismatch` | Use `-a amd64` or `-a arm64` to specify target arch explicitly |
| OCI pull errors | Verify registry credentials with `uds zarf tools registry login` |
| Package deploy timeout | Set `timeout` on the package in `uds-bundle.yaml` (e.g., `timeout: 10m`) |

To retrieve logs from the most recent operation:

```bash
uds logs
```
