UDS CLI is the primary interface for interacting with components in the UDS ecosystem. It streamlines the deployment of mission applications and secure infrastructure by bundling multiple Zarf packages into a single deployable artifact — a **UDS Bundle**.

Beyond deployment, UDS CLI includes a built-in task runner ([UDS Runner](/cli/how-to-guides/use-uds-runner/)) for automating workflows and a `monitor` command for observing cluster state in real time.

---

## Key Commands

| Command | Description |
|---|---|
| `uds create` | Build a UDS Bundle from a `uds-bundle.yaml` |
| `uds deploy` | Deploy a bundle from a local file or OCI registry |
| `uds inspect` | Inspect bundle metadata, images, SBOMs, and variables |
| `uds publish` | Publish a bundle to an OCI registry |
| `uds remove` | Remove a deployed bundle from a cluster |
| `uds run` | Execute tasks from a `tasks.yaml` via UDS Runner |
| `uds monitor pepr` | Stream Pepr admission and operator logs |
| `uds zarf` | Access the vendored Zarf CLI |

Full command reference is available in the [uds-cli GitHub repository](https://github.com/defenseunicorns/uds-cli/tree/main/docs/reference/CLI/commands).

---

## Schema Validation & IDE Setup

UDS CLI ships with a JSON schema for `uds-bundle.yaml`. To enable IDE autocompletion and validation, see the [schema validation docs](https://github.com/defenseunicorns/uds-cli/blob/main/docs/reference/CLI/schema-validation.md) in the repository.
